=== The selected element would in some situations, when it was reused or otherwise reset not have an initial selected value when expected to.
//#TINY-9679

In previous version of {productname}, the browser displayed difficulties while switching to a elements child, which lead to it defaulting to an empty state.

To resolve this issue, element such as this can no longer be empty unless the user explicitly sets it as such.

In {productname}, these elements will now have a populated value unless the user specifically chooses an empty selection.

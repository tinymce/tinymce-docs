The following premium plugin updates were released alongside {productname} 6.5.

=== Word count was inaccurate for documents with specific characters.
//#TINY-8122

In previous versions of {productname}, the word count function in certain documents was inaccurate due to a bug. This bug occurred when specific characters, such as `+^, â„–, ~, +, |, `+` and `+` were treated as separate words instead of being recognized as part of the word they were attached to.

As a consequence, the word count function provided incorrect counts, which may have resulted in errors regarding the final word count for the document.

In {productname} 6.5, the word count function's Unicode punctuation string has now been updated to include these characters as exceptions. By doing so, the updated function now accurately count words, including the previously mishandled characters, ensuring a more reliable word count in the users final document.

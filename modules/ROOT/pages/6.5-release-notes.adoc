=== It was possible to apply case change to `contenteditable="false"` elements.
//#TINY-9466

In previous versions of {productname}, selecting content inside a `contenteditable="false"` section while having the casechange plugin active made it possible to apply the casechange features via the toolbar buttons and menu items.

As a result in {productname} 6.5, the casechange toolbar button and menu items will now be disabled when the selection is in `contenteditable="false"` section.

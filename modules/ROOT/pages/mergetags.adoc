The {pluginname} plugin allows the user to insert a merge tag (also known as a personalization token, or a mail merge field) from a drop-down list when a specified prefix is typed.

Once a merge tag is inserted, the plugin leaves a non-editable variable wrapped with a prefix and suffix, making it easily identifiable.

== Basic setup

[source,js]
----
tinymce.init({
  selector: 'textarea',  // change this value according to your HTML
  plugins: 'mergetags',
  toolbar: 'mergetags',
});
----

The {pluginname} plugin provides an xref:autocompleter.adoc[autocompleter] for adding a merge tag without using the toolbar button or menu item.

The autocompleter is triggered by typing a specified prefix, controlled by the xref:#mergetags_prefix[`+mergetags_prefix+`] option.

By default, this prefix is `{{`.

Entering characters after the prefix begins filtering the merge tags list.

== Usage notes

. Merge tag are non-editable but can be inline-formatted.
+
For example, a merge tag can be set to any available typeface and type-size and can be set to *bold*, or _italic_.

. Merge tags can be changed.
+
Select the Merge tag, click the Merge tag toolbar button, and select a new tag from the menu that presents.

Users can paste from clipboard merge tags, which are not available in mergetags_list.
User can customize prefix and suffix styles by tweaking provided mce-mergetag-affix class.
Users can simplify merge tag menu navigation by grouping merge tags in categories and providing user-friendly titles.

[[uploadcare-supported-files]]
== `uploadcare_supported_files`

Defines the supported file types and their extensions for upload.

*Type:* `+Array<Object>+`

*Default:* See supported file types below

=== Configuration Schema

Each entry must contain:

[cols="1,1,1"]
|===
| Property | Type | Required

| `mimeType`
| String
| Yes

| `extensions`
| Array of Strings
| Yes
|===

=== Default Supported File Types

The uploadcare plugin supports a comprehensive range of file types by default, including images, videos, documents, and audio files.

.Example: Basic supported file types configuration
[source,js]
----
tinymce.init({
  selector: 'textarea',
  plugins: 'uploadcare',
  toolbar: 'uploadcare',
  uploadcare_public_key: '<your-public-key>',
  uploadcare_supported_files: [
    // Image files
    { 
      mimeType: 'image/jpeg',
      extensions: ['jpg', 'jpeg', 'jpe', 'jif', 'jfi', 'jfif']
    },
    { 
      mimeType: 'image/png',
      extensions: ['png']
    },
    { 
      mimeType: 'image/webp',
      extensions: ['webp']
    },
    // Video files
    { 
      mimeType: 'video/mp4',
      extensions: ['mp4']
    },
    { 
      mimeType: 'video/webm',
      extensions: ['webm']
    },
    // Document files
    { 
      mimeType: 'application/pdf',
      extensions: ['pdf']
    },
    { 
      mimeType: 'application/vnd.openxmlformats-officedocument.wordprocessingml.document',
      extensions: ['docx']
    },
    // Audio files
    { 
      mimeType: 'audio/mpeg',
      extensions: ['mp3']
    }
  ]
});
----

=== Notes

* The `uploadcare_supported_files` option allows customization of which file types can be uploaded through the uploadcare plugin.
* When this option is not specified, the plugin uses the comprehensive default list shown above.
* The plugin validates both MIME types and file extensions to ensure compatibility.
* Image file types are also influenced by the `images_file_types` option, which can extend the supported image formats.
* Video files support additional properties through the `uploadcare_video_properties` option.
